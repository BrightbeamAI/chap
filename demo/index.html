The Collaborative Human-Agent Protocol

The room where humans and agents do real work.

CHAP is the open standard for multi-human, multi-agent collaboration. Third pillar of the agent protocol stack — alongside MCP and A2A — for what happens when humans, agents, and tools work together in one shared workspace.

The problem · told as a story

Tuesday afternoon at TechCorp Support.

Maya, a senior support agent, opens her review queue. Forty-seven customer responses are waiting — drafted overnight by an AI agent. She'll approve most. A few she'll reject. Some she'll edit before sending.

47
drafts in Maya's queue this afternoon
300
drafts she'll review by Friday
15,000
drafts she'll review by year-end

Here's the question that pays for CHAP:

what if every one of those edits became data?

In most companies today, they don't. Maya edits the AI's draft, sends the response, and moves on. The edit lives in nobody's database. Nobody learns from it. Next week the AI makes the same mistake. Maya edits it again.

Repeat for a year, and you've thrown away the world's richest fine-tuning dataset — the one your humans built for free by doing their normal job.

CHAP exists to capture that signal. Not as a side feature. As the core wire format. Every override is structured (diff, rationale, tags, policy references) and lives in the audit log forever. Queryable. Aggregatable. Trainable on.

The analogy

It's track changes for AI.

The same shape an editor uses when marking up a writer's draft — structured for machines as well as for humans.

What Maya sees

We sincerely apologise for the extreme delayjust wanted to follow up — your order is in transit and will arrive within the next three business days. We deeply regret any inconvenience caused.Let me know if you don't see it by then.

Tone too apologetic for a routine tracking question
What CHAP records
"method": "decide.override",
"params": {
  "task_id": "tsk_…",
  "diff": [
    { "op": "replace",
      "path": "/body",
      "value": "Thanks for following…" },
    { "op": "replace",
      "path": "/tone",
      "value": "warm_professional" }
  ],
  "rationale": "Over-apologetic for routine",
  "tags": ["tone-softened"],
  "policy_refs": ["support-tone-v2"]
}

The track changes on the left are what Maya naturally does. The JSON on the right is what the protocol records. Same shape, different audience.

JSON Patch (RFC 6902) isn't new. Rationale fields aren't new. Tags aren't new. What's new is that they live together as a typed protocol primitive, captured automatically as part of the review flow.

The protocol

Three pillars. One stack.

The open agent protocol stack has been forming over the last two years. CHAP is the layer where humans live.

CHAP THIS LAYER
Humans + agents collaborating in workspaces
A2A
Agents delegating to agents
MCP
Agents calling tools

Each protocol owns one concern.

They compose without conflict. An agent inside a CHAP workspace can call MCP tools and delegate to A2A peers. The CHAP audit log cites those interactions without re-implementing either protocol.

Core in seven methods

CHAP Core is the minimum — implementable in a weekend in any language. Seven methods over JSON-RPC 2.0. That's enough to run a compliant workspace.

workspace.describe
What is this room? Who's in it? Which profiles are advertised?
participant.join
Add a human, agent, or service to the workspace.
participant.leave
Remove a participant.
task.create
Delegate work to an assignee with structured input.
task.update
Report progress; transition state.
task.complete
Deliver output and finalise the task.
audit.read
Query the workspace's history with filters and ranges.

Everything else — structured review, multi-party deliberation, mode promotion, signed audit, identity binding — is an optional profile layered on top. You pay for what you use.

The killer move

Anatomy of an override.

When Maya edits the bot's draft, four things get recorded together. Every one of them is queryable structured data — no separate tagging step, no manual logging, no afterthought.

Override artefact art_01HZ9YX7K3X8M2V4N6P8R0T3B
diff
What changed · RFC 6902
replace /body
replace /tone → "warm_professional"
replace /severity → "low"
rationale
Why · free text
Tone was over-apologetic for a routine tracking enquiry. Severity downgraded since the order is in normal transit.
tags
Aggregation keys · controlled vocab
tone-softened severity-downgraded length-reduced
policy_refs
Anchors · external guidelines
support-tone-guideline-v2 severity-rubric-v3
Reviewer human:maya@techcorp.example Applied to tsk_01HZ9YX7K3X8M2V4N6P8R0T3B Timestamp 2026-05-12T14:38:22.481Z

None of the four fields is novel on its own. The novelty is that they live together as a single typed protocol object — captured at the moment of review, preserved in the audit log forever, available to anyone with audit access.

Think of it as double-entry bookkeeping for AI oversight. Each override has both what changed and why, locked together at the time of the entry. You can't have one without the other; the protocol won't let the entry through.

What this tells you

The agent is getting the tone wrong.

tone-softened dominates at 52% of overrides. The agent is over-apologetic for routine queries. One-line prompt change away from a fix.

Secondary signal: severity classification skews high. The rubric needs more low-severity examples.

This report came from nothing but the audit log. No analytics instrumentation. No retroactive labelling. Every data point fell out of Maya doing her job.

That's the dividend. By Wednesday next week, the prompt is updated. By Friday, the tag distribution starts shifting. By the end of the next sprint, the agent is measurably better at the thing Maya was already correcting it on.

The protocol made the loop visible. The protocol kept the loop short.

Adoption

Pick what you need. Skip what you don't.

CHAP is Core plus eleven optional profiles. A working deployment starts with Core alone. Each profile adds a focused capability and stays independent of the rest.

Start here 2 profiles
core/1.0 Mandatory
7 methods. Workspaces, participants, tasks, audit log. Weekend build.
review/1.0 Killer feature
request, approve, reject, override, abstain, escalate. The one you just saw.
Workflow 5 profiles
modes/1.0
Shadow → trial → production. Roll out new agents safely.
routing/1.0
Cost, criticality, confidence-driven routing. Carry signals, decide depth.
deliberation/1.0
Quorum, weighted votes, vetoes. Multi-party decisions.
whisper/1.0
Deadline-bound interrupt questions with default-if-lapsed.
handoff/1.0
Shift changes; group-routed handoffs.
Identity & trust 4 profiles
identity-oidc/1.0
OIDC binding with cnf.jwk or DPoP. Reuse your existing identity layer.
identity-vc/1.0
W3C Verifiable Credentials. For regulated or cross-org identity.
security-signed/1.0
Ed25519 + JCS canonicalisation. Per-message non-repudiation.
audit-scitt/1.0
IETF SCITT transparency log. Offline-verifiable receipts.
Operations 1 profile
control/1.0
Pause, resume, snapshot, rollback. Operational control plane.
Standards reused
JSON-RPC 2.0, RFC 8785 (JCS), RFC 6902 (JSON Patch), OIDC + RFC 7800 (cnf.jwk), W3C VC 2.0, IETF SCITT, COSE (RFC 9052), SPIFFE, in-toto. The only protocol-level inventions are the methods themselves and the override-with-rationale shape. Everything else is composition.
